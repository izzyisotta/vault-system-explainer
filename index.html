The one idea

Borrowed from Freddie's system, adapted to my vault.

Don't file into folders. File into metadata, and let a saved query do the cross-referencing.

Folders force one note into one place, so a conversation that touches three projects has to pick a home and the tree fights you. Metadata and links don't have that problem: a note can belong to everything at once, surfaced wherever it's relevant. Capture stays brainless so it never becomes a chore; all the intelligence happens later, in the vault, where I control it.

Capture is cheap

Record or share with zero decisions. No categorising in the moment.

Vault does the work

Enrichment, tagging and routing happen on my side, not at capture.

Queries draw the map

Tag well once; dashboards and related-notes rebuild themselves.

How the whole vault is structured

Folders where structure is load-bearing; flat + metadata everywhere else.

Stays in folders

  • INSEAD — course → lecture → Sources/L##/. The lecture-note pipeline depends on this exact shape. Time-bounded: it freezes as an archive at graduation.
  • Projects (startups) — a project is a folder.
  • _Setup / infrastructure — templates, skills, config.

Flat + metadata

  • People and Inbox — already flat databases driven by Bases. Live
  • Content Library — one flat folder, classified by content_format + landing pages. Live
  • Meetings / talks — the next thing to go flat. Planned
  • Creative / Career — flatten later, by metadata.
The hierarchy is a link graph, not a folder tree. Going forward, INSEAD is the one permanent exception; everything new is flat and metadata-first.

How tagging works

Three kinds of metadata, three different mechanisms. This is what makes the queries possible.

Entity → link

People, companies, concepts — things with their own note. Stored as [[wikilinks]]. Gives backlinks + graph + one canonical record.

Attribute → field

Date, type, status, area, interest, duration — values that describe the note. Stored as frontmatter fields. Sorted/filtered in Bases.

Keyword → tag

Genuine loose keywords + provenance (voice-transcript). Tags stop being the dumping ground for everything.

The controlled vocabularies (closed lists, never free-form)

Areas: insead-classworkinsead-otherai-workflowstartupscareernetworkingpersonal-socialcreative-ideasother

Interests: aiconsciousnessrationality-cognitive-scienceeconomics

Rule of thumb: does this thing have its own note? Yes → link it. No, it describes the note → field. Only loose keywords → tag.
